◆ Cy_MXOTPC_WriteRowReadBack()

cy_mxotpc_status_t Cy_MXOTPC_WriteRowReadBack ( uint32_t  otpRowNum,
uint32_t  otpWriteData,
uint32_t *  otpReadBackData,
bool  enable_ecc 
)

Writes a particular row with 32-bit data with/without ECC and reads the row back.

Indirect PROGRAM Sequence 1.Write Following Bit OTP_CTL.OTPProg_En =1 // Enable OTP programming at top level control 2.Check whether OTP is busy or not OTP_CMD.START == 0 // OTP is not busy 3.Check whether programming is enabled or not (PROGRAM Enable Seq) OTP_STATUS.ProgOK == 1 // OTP programming enabled If this bit is not set, perform following steps, otherwise skip. a.OTP commands to enable OTP Programming i.Write OTP_PROGDATA = 0xF ii.Write OTP_CMD reg (with CMD =0x02), VALUE: 0x9100_0000 iii.Wait for OTP_CMD.START == 0 iv.Repeat from point i to iii for OTP_PROGDATA 0x4, 0x8 and 0xd in sequence Check whether programming is enabled or not OTP_STATUS.ProgOK == 1 // OTP programming enabled 4.Write OTP_PROGDATA register Write 32-bit data value to be programmed inside OTP_PROGDATA reg 5.WRITE OTP_CMD Register Write following value in OTP_CMD reg. For OTP_CMD command (0x0A) at 511th row, you may use following value, 0x8501FF00 6.check completion of access Read 31st bit of OTP_CMD register to check completion of programming. OTP_CMD.START == 0 // access completed

  1. Check programming status OTP_STATUS.PROGFAIL == 1 // Program failure

    Parameters
    otpRowNumContains the Row Number to be written
    otpWriteDataContains the Data to be written
    otpReadBackDataContains the Read back Data of the Row Number passed.
    enable_eccContains boolean, which tells to enable ECC for that row or not.
    Returns
    the MXOTPC API status cy_mxotpc_status_t.

◆ Cy_MXOTPC_IndirectReadBit()

cy_mxotpc_status_t Cy_MXOTPC_IndirectReadBit ( uint32_t  otpRowNum,
uint32_t  bitNum,
uint32_t *  otpReadBit 
)

Reads a bit status of a particular OTP Row using Indirect access.

Indirect READ Sequence

  1. Check whether OTP is busy or not OTP_CMD.START == 0 // OTP is not busy
  2. WRITE OTP_CMD Register Write following value in OTP_CMD reg. For READ command (0x00) at 15th row and 3rd bit, and clearing previous read error if exist, you may use following value, 0X9000_0F03
  3. check completion of access Read 31st bit of OTP_CMD register to check completion of programming. OTP_CMD.START == 0 // access completed

  4. Check Read status & Read bit OTP_CMD.ReadVal // Read value from specified bit. OTP_CMD.RD_ERR == 1 // Error in bit reading Read ECC_STATUS register for ECC status of the row. More details on ECC refer to register description.

    Parameters
    otpRowNumContains the Row Number, whose bit status should be Read
    bitNumContains the bit position to be Read
    otpReadBitContains the Read bit of bitNum position from the Row Number passed.
    Returns
    the MXOTPC API status cy_mxotpc_status_t.
